# `manifest_gois.json`（schemaVersion 3）

## 収集ソース（Wikidot）

- 取得元: `https://scp-jp.wikidot.com/goi-formats-jp` の `#page-content` 内、`.content-panel` 優先（無ければ `#page-content` 直下を走査）

## 分類

| 区間 | 検出 | `goiRegions` キー |
|------|------|-------------------|
| 要注意団体-EN ブロック | `h1` 文言に `要注意団体-EN` を含む | `en` |
| 要注意団体-JP ブロック | 同上 `要注意団体-JP` | `jp` |
| 上記以外で `要注意団体-` で始まる `h1`（`インフォメーション` 除外） | 例: `要注意団体-CN` など。細分化はせず **1 束** | `other` |

## 団体（1 エントリ = 1 `h2`）

- 直前の兄弟 `<p><a name="...">` があれば、その `name` を団体 `i`（なければ `h2` 由来の ASCII 仮 `id`）
- `h2 > span` に `a[href]` があり、**同一 Wiki の単一スラッグ**に解決できる場合のみ、団体行として扱い `t`（表示名）と `u`（団体ハブ URL）を書き込む
- 団体ハブに解決できない行（テキストのみ、不正パス、外部のみ等）: **配列に含めない**（JSON にも出さない）
- 子記事列（`ul` 内リンク等）は **収集・出力しない**（フラット `entries` も空）

## トップレベル

- `schemaVersion`: `3`（他マニフェストの `2` とは独立）
- `entries`: **空配列** `[]`（従来の子記事フラットは廃止）
- `metadata`: **空オブジェクト** `{}`
- `goiRegions`: `{ "en" | "jp" | "other": [ 団体オブジェクト, ... ] }`
- 団体オブジェクト: `{ "i", "t", "u" }`（`u` は必ず団体ハブの絶対 URL 文字列）

## 互換

- 旧 **schema 2**（`goiRegions` なしのフラットのみ）に対応したクライアントは、従来のフラット一覧にフォールバック可能。
- 古い v3（団体に `entries` 配列があった版）の JSON を読むクライアントは、未知キー `entries` を無視すればよい。
